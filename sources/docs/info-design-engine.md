# I envision an "information design pattern library" as one of Report Kitchen's key assets

This library would list items like those identified in info-design-goals.md: 

- accordions
- horizontal tabsets
- callout boxes (..etc..)

And for each would contain (as information for humans and/or agents as appropriate):

- Educational content: 
  - what is it, what's it good for, when/how to use it, etc.
  - Examples
  - Variations, options, extensions, customizations, etc.
- How to identify the pattern in an existing work
- How to identify content in a body of work where this pattern *could* be used or recommended
- How to measure the use and/or assess the success of the pattern, once its in use in one of our documents

## How do we achieve this?

I belive compiling this -- especially how to identify the pattern in use and how to identify content approprite to reformat using the pattern -- manually, will be slow and incomplete.

I believe we should consider investing in creating this engine such that it can learn from documents we feed it, and create the processes itself.
