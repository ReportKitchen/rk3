# Random notes



## Front Matter
Everything from the cover first page up until the beginning of the real content, is front matter, and is generally handled much differently in print vs web.  Much of the difference is document-dependent and a user decision.

### Cover
A big challenge: Take a vertical, precisely designed cover, and reformat it to be horizonal, responsive, and (in most cases) a header only, as opposed to the full page.

RK3 will offer multiple Home Page templates for the user to choose from as a starting point.  They're mostly different ways of presenting the same information, and require the same content to be extracted from the source document's cover page.

## User options

### Create callouts for pages with background color or image
If a PDF page is entirely one background color, or consists of an image background with text on top, then 

- default: create a new callout (unless the style guide already has one with that background color) encompassing the content of that PDF page, using the colors on that page. If the page has an image background use that as the background for the callout.

- no callout: remove the color and/or image background, realign the text/header/link colors to the document's defaults.
